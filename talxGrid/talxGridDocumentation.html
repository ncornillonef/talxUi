Easy, quick, powerful data grid. Supports sorting, searching, and paging. Handles over 50K rows. Simply give a target element, and options with a column definition and the data to display.

Overview

Make the selected element a talxGrid widget. Semantic requirements:

The markup of your talxGrid container needs a containing element, and options with column definitions and rows.

<div id="talxGrid1">
    This text is replaced with data defined in the options.
</div>

Handles 50,000 rows effortlessly. By 100,000 rows, sorting and searching are a tiny bit slow (still less than a second). Wow, that's fast!

Dependencies

  • jQuery
  • jQuery UI

Options

Required options have a * after them.

  • columns*

    Type:
    column object array
    Default:
    none

    columns is a required option to initialize correctly.

    This is a JSON array of column definitions. A column definition is an object with a header, and a field.

    column object

    header (required)
    [""] Display used for the column header
    displayField (required)
    [""] string property/field name
    allowSort
    [true | false]
    allowFilter
    [true | false]
    allowFilterSelect
    [true | false]
    dataType
    [string | int | float | date | boolean]
    displayMask
    ["{0}"] string used to format the display field. Only supports {0} at the moment (not additional formatting of {0} within the braces). {0} is the value of displayField. {1} is the row index.
    linkField
    [null (string)] if a link is desired, the field used in the link mask. A null value indicates no link is shown.
    linkMask
    ["{1}"] if a link is desired, the masked used to format the link. Only supports {0} & {1} at the moment (not additional formatting within the braces). {0} = linkField, {1} is the result of displayField formatted with displayMask. {2} is the row index.
    tdClass
    [""] an additional CSS class to add to the td cells in this column.
    width
    ["*"] This is the width of the column. Default is auto.

    Code examples

    Initialize a talxGrid with the columns option specified.
    $( ".selector" ).talxGrid({ columns: 
    { header: "Name", displayField": "userName" },
    { header: "E-Mail Address", displayField: "email"}
]});
    Get the columns option, after init.
    //getter
var columns = $( ".selector" ).talxGrid( "option", "columns" );

    This is a JSON array of the data objects in the table. This is the complete list of items, even if it is more than one page's worth. The field names in columns is used to bind the data in the column.

Methods

  • destroy

    Signature:
    .talxGrid( "destroy")

    Remove the talxGrid functionality completely. This will return the element back to its pre-init state.

  • disable

    Signature:
    .talxGrid( "disable")

    Disable the talxGrid.

  • enable

    Signature:
    .talxGrid( "enable")

    Enable the talxGrid.

  • items

    Signature:
    .talxGrid( "items")

    This is the base list of data used to drive the table. (Same as "option", "data".)

  • option

    Signature:
    .talxGrid( "option", optionName, [value] )

    Get or set any talxGrid option. If no value is specified, will act as a getter.

  • option

    Signature:
    .talxGrid( "option", options)

    Set multiple talxGrid options at once by providing an options object.

  • pageIndex

    Signature:
    .talxGrid( "pageIndex", index)

    Set the current page on the grid. Remember the first page on the grid is 0.

  • refresh

    Signature:
    .talxGrid( "refresh")

    Refreshes the current page of the talxGrid. Best used after editting a row of data.

  • reset

    Signature:
    .talxGrid( "reset")

    Resets the talxGrid's search criteria (clears the filter). This is like the user pressing the Reset button/link.

  • rowData

    Signature:
    .talxGrid( "rowData", index)

    Returns the data item backing the row of data represented by the index. The index is relative to the page, not the root item list. For example, the first row on the page has an index of 0 (no matter what the current page is).

  • search

    Signature:
    .talxGrid( "search")

    Triggers a search using the current search criteria. This is like the user pressing the Search button/link.

  • select

    Signature:
    .talxGrid( "select", criteria)

    Finds the first matching row in the data, switches to that page, and adds the CSS class of "ui-state-selected" to the row.

    criteria is an object with two members, "field" and "value". "field" is the name of a field on the data object, and "value" is the value to match.

  • widget

    Signature:
    .talxGrid( "widget")

    Returns the .ui-talxGrid element.

Theming

The talxGrid plugin uses the jQuery UI CSS Framework to style its look and feel, including colors and background textures. We recommend using the ThemeRoller tool to create and download custom themes that are easy to build and maintain.

If a deeper level of customization is needed, there are widget-specific classes referenced within the talx-ui.css stylesheet that can be modified.
